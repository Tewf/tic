Partie1: Résume de chapitre 1

Les techniques de l'information et de la communication (TIC) regroupent l'ensemble des outils utilisés pour manipuler, traiter, produire, faire circuler, transmettre, modifier et échanger des ressources et moyens de transmission et d'échange. Les TIC sont présentés dans 3 catégories : téléinformatique, télécommunication, électronique et multimédia. Les objectifs recherchés sont de la connectivité et d'une meilleure diffusion de l'information. Modularité, implantation simple, utilisation facile, fiabilité, composantes de TIC matérielle et logicielle. Caractéristiques: rapide, décentralisation du pouvoir, mondialisation, répartition de l'information et intensité de transmettre et recevoir des informations à n'importe quel moment et région. La communication est un outil indispensable pour tous les organismes et a des objectifs suivants : prendre conscience, modifier ou renforcer l'image de produit, modifier les comportements en provoquant l'achat, l'essai. Le système de communication comprend les concepts suivants : les partenaires de communication, le récepteur, les vecteurs, les fonctions, le codage, la décodage, la réponse, la feedback, et le bruit. La communication institutionnelle (corporate) consiste à communiquer sur l'image de l'entreprise ou de l'institution. La communication commerciale ou de produit consiste à mettre en avant les caractéristiques, les performances et les avantages du produit dans une communication crédible et cohérente. Les schémas de communication : la communication interpersonnelle, la communication de groupe, la communication de masse et le média et le multimédia.

1) Les composants matérial d’un Réseau informatique: Les cartes réseau: aussi appelées cartes d’interface réseau (NIC), Les cartes réseau sont installées à l'intérieur de l'ordinateur et sont impedance de communication entre ordinateurs et données via Internet. SWITCH: un switch (commutateur réseau) est un composant matériel utilisé pour connecter des appareils à un réseau local (LAN). il permet à plusieurs appareils (des ordinateurs, des imprimantes, des serveurs, etc..), de communiquer entre eux en utilisant des adresses MAC (Media Access Control). Les switches utilise cette table pour envoyer les données uniquement à l'appareil destinataire. Ils sont disponibles en différentes tailles, allant des petits switches pour les réseaux domestiques, aux switches de niveau entreprise pour les réseaux plus importants. HUB: un hub est un composant matérial utilisé pour conecter plusieurs appareils sur un réseau local (LAN), il reçoit les données envoyées par un appareil et les transmet à tous les autres appareils connectés. Les hubs ont été largement remplacés par des switches, qui sont plus efficaces pour la gestion du trafic réseau et permettent une communication plus rapide et plus sécurisée entre les appareils connectés. Les hubs sont encore utilisés dans des applications spécifiques ou dans des réseaux de petite taille où la complexité d'un switch n'est pas nécessaire 2) Le Modèle OSI: Le Modèle OSI est modèle de référence pour communication réseau car il divise les communication en 7 couches qui sont les suivants : 1-La couche physique 2-la couche liaison de données 3-La couche réseau 4-La couche transport 5-La couche session 6-La couche présentation 7-La couche application c'est une référence qui n'est pas utiliser dans le réseau modèrne 3) Le Modèle TCP/IP: c'est un modèle de référence pour les communications de réseau, qui est utilisé pour la conception et la mise en oeuvre des réseaux informatique modèrnes. Le modèle TCP/IP divise les communications en quatres couches: 1-la couche liaison de données 2-la couche réseau 3-la couche transport 4-la couche application 4) L'adressage : chaque communication dans le réseau a ses propres règles/ Adressage: définit les deux unités Architecture:qui définit les roles endossés par les deux unités Protocoles: qui assurent l'échange des données IP: il s'agit de l'adresse de protocole internet qui est le numéro qui chaque ordinateur des réseau aux quels il apportient et il est de deux types :IPV4 et IPV6 -L'adress IPV4: il se compose de 4 bits dont la première partie est pour l'adress réseau et la seconde pour l'hote. l'adressage de IPV4 se fait comme suite: l'adresse réseau , l'adresse de diffusion , l'adresse hotes ,le masque de réseau et de sous réseau

Un page Web est interprétée et affiché par un navigateur et est localisée sur Internet à l'aide d'une adresse (URL). Un site web est une regroupement de pages Web autour d'une thématique commune reliées entre elles par des liens hypertextes. Lorsqu’on démarre un dépôt Git en important un répertoire déjà existant depuis notre machine, les fichiers sont au départ tous non suivis. Lorsqu’on clone un dépôt Git déjà existant, c’est différent puisqu’on copie tout l’historique du projet et donc les fichiers sont tous déjà suivis par défaut. Lors du démarrage d’un dépôt Git à partir d’un dépôt local, on demande à Git de valider l’ensemble des fichiers du projet. Un fichier est "validé" lorsqu’il est stocké dans la base de données locale. Lors du clonage d’un dépôt déjà existant, les fichiers sont enregistrés par défaut en base et ils sont donc validés par défaut. Pour initialiser un dépôt Git, on utilise ensuite la commande git init comme ci-dessous. Cela crée un sous répertoire .git qui contient un ensemble de fichiers qui vont permettre à un dépôt Git de fonctionner. Lorsqu’on utilise git init, Git nous renvoie un message en nous informant que le dépôt Git a bien été initialisé et qu’il est vide. C’est tout à fait normal puisque nous n’avons encore versionné aucun fichier . A ce niveau, nous avons donc d’un côté notre projet contenant un ensemble de fichiers et de ressources sur lesquelles on travaille ainsi qu’un dépôt Git qui sert à gérer les différentes versions de notre projet. A chaque fois qu’on souhaite enregistrer une modification de fichier ou un ajout de fichier dans le dépôt Git, on va devoir utiliser les commandes git add et git commit comme on a pu le faire dans la leçon précédente.
